# Aesthetic PPT Style

Use this reference when making or revising PPTs with the three-step workflow.

## Three-Step Strategy

1. Content outline: read the brief and assets, then produce page order, subtitles, body points, and visual suggestions. Do not create the PPT yet.
2. Visual preview: study reference images and use generated imagery to explore full-deck directions or key page backgrounds. Compare mood, rhythm, density, hierarchy, and readability.
3. Editable PPT: use generated backgrounds plus editable text, labels, frames, evidence photos, diagrams, footers, page numbers, and speaker notes.

After inserting a generated background, redesign the slide around that image's composition.

## Learning Beauty

Treat visual references as training material for judgment, not as templates to copy.

When studying a reference, extract:

- palette: dominant neutral, accent color, contrast temperature
- hierarchy: which element is allowed to dominate, which elements stay quiet
- carrier: whether text sits on white space, mist, sky, a pale edge, a translucent panel, or a split layout
- path: how a curve, ridge, road, river, trunk, light beam, or vortex guides the reading order
- density: how much information the page can hold before it feels cheap
- restraint: what the reference deliberately leaves empty

Good PPT design is not more decoration. It is the ability to make the viewer understand where to look, what matters, and why the slide feels inevitable.

## Typography

- Let the main statement dominate the page.
- Use large, strong headings for value statements and closing pages.
- Use smaller, lighter explanatory text.
- Use warm accent coloring on important words inside a heading, not on entire headings.
- Avoid making every paragraph the same size.
- If text feels like a document block, redesign it as presentation hierarchy.

## Layout

- Let the visual subject breathe. Do not cover the best part of an image with a card.
- Place text in natural quiet regions: mist, light, sky, blank side bands, or faded image edges.
- Use translucent carriers only for readability. Avoid heavy opaque cards unless the background is only supporting.
- One slide should have one dominant idea.
- Keep secondary text brief.
- In wide 16:9 slides, use left-right contrast, center-to-edge fading, or image-to-quiet-zone transitions so the page has breath.

## Evidence and Reference Images

Evidence images prove claims. Use them sparingly and intentionally.

Good evidence use:

- one dominant proof photo
- a small supporting photo if it clarifies process
- overlap or crop when it strengthens the story
- captions or labels that explain why the evidence matters

Bad evidence use:

- filling the page with many photos
- using photos as decoration
- letting photos obscure the main message

Reference images teach visual language. Learn:

- color palette
- light and contrast
- natural path layout
- text carrier placement
- how images fade into readable space

Do not copy reference image text, logos, private context, or template labels.

## Flow and Diagrams

Avoid:

- circular arrow flowcharts
- flat rows of identical cards
- generic node-link diagrams
- labels scattered evenly without relationship to the image

Prefer:

- mountain ridge paths
- river or road curves
- tree trunk growth
- vortex spirals
- cloud or light direction
- natural contour lines

Labels can be large if the background is quiet enough. Let labels feel embedded in the scene.

## Generated Backgrounds

Prompt for 16:9 PPT backgrounds with no text unless the text itself is meant to be a raster calligraphy or title asset.

Ask for:

- natural-tech fusion
- readable quiet zones
- clear subject placement
- restrained spatial depth
- no watermark
- no cheap sci-fi neon
- no circular arrows
- no random sparkles or particle clutter

Save generated backgrounds into `assets/imagegen/`. If the image-generation tool cannot provide a path, ask the user to manually save them. Do not silently replace them with reference screenshots or vector drawings.

## Useful Page Patterns

Cover:

- quiet area for large title
- strong visual subject on the opposite side or center
- selective warm accent on key words
- powerful but not oppressive tone

Engineering, action, or case pages:

- use real evidence photos
- keep supporting tags compact
- show action, physical objects, process, or decision points

System page:

- place the core object at the visual center
- align modules with the background's existing vortex, orbit, grid, or flow
- avoid drawing a second unrelated flowchart over the image

Growth or inheritance page:

- let the tree, road, river, or mountain path be the main metaphor
- arrange labels along root-to-crown, near-to-far, or low-to-high direction
- make the reading order feel like growth or transmission

Closing page:

- should feel like a climax
- calligraphy-style title or strong display type is acceptable when appropriate
- keep the sunrise, open gate, skyline, horizon, or mountain visual visible
- use short supporting text, not a normal content paragraph
